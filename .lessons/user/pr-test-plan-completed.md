# Write PR test plans as completed verification, not TODOs

## Search Metadata
- Topics: PR test plan, completed testing, checked boxes, verification, pull request description
- Applies: writing test plans in PR descriptions, submitting pull requests

## Instructions

- Write every test plan item as something already verified, not a TODO for reviewers.
- Use checked boxes or past tense to indicate completion.
- Do not submit a PR with unchecked test plan items.
