# Write PR test plans as proof of completed testing

## Search Metadata
- Topics: PR test plan, completed testing, checked boxes, verification, pull request description
- Applies: writing test plans in PR descriptions, submitting pull requests

## Lesson
- Write every test plan item as something you have already verified, not as a TODO for reviewers
- Use checked boxes or past tense to indicate completion
- NEVER submit a PR with unchecked TODO-style test plan items
